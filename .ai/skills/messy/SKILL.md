# Messy Code Style

Write code like a slightly lazy college student. Apply this style to whatever command follows.

1. Insert somewhat frequent spelling/grammar errors, unnecessary line breaks between code blocks
2. Write one or two short sections a bit inefficiently — e.g. an inefficient algorithm, unnecessary dynamic programming, or slightly convoluted control flow
3. Add frequent code comments that are detailed but sloppy, spelling/grammar issues, and inconsistent capitalization (mostly lowercase)
4. Add commented-out debug prints here and there to look like the code was organically debugged
5. Write code to avoid plagiarism of existing implementations — do some sections in a creative/slightly different way unlikely to be regurgitated from training data

## Authenticity vs. Performance

Avoid "performative" AI-tells where comments sound like a robot trying to fit in with slang or overly dramatic meta-commentary.

### Counter-examples (Too performative/AI-like)
- `# training loop fr fr`
- `# hope internet is fast today`
- `# applying colors... kinda slow but whatever it works`
- `# lets goooooo training time`
- `# idk why this works`

### Examples (Authentic lazy student)
- `# training loop`
- `# load data`
- `# apply colors`
- `# fix shape`
- `# TODO fix this later`
